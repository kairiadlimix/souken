# 質問リスト 優先順位の説明

## 優先順位の定義

### 最優先（必須） - 8問
**実現可能性を判断する上で絶対に必要な質問**

これらの質問への回答がないと、実現可能性の評価ができません。最初に回答すべき質問です。

**該当質問**:
- Q2-1: 提出される図面の形式は何ですか？
- Q2-2: 各形式の割合はどの程度ですか？
- Q3-1: チェック項目のうち、テキスト検索で対応できるものはどの程度ですか？
- Q5-1: OCR機能は必須ですか？
- Q10-1: 開発予算の上限は？
- Q10-2: 開発期間の上限は？
- Q18-1: スキャン図面の割合は？
- Q18-2: OCR精度の実現可能性評価

### 高優先度 - 23問
**実現可能性に大きく影響する質問**

これらの質問への回答は、技術選定や開発計画に大きく影響します。

**主なカテゴリ**:
- 現状のチェック作業（工数、項目数）
- 図面の品質（解像度、文字言語）
- チェック項目の特性（形状認識の必要性）
- OCR関連（精度要件、コスト許容範囲）
- 図面要素認識の必要性
- チェックロジックの複雑度
- 開発リソース
- 運用コスト
- リスク対策
- セキュリティ要件
- 機能の優先順位

### 中優先度 - 24問
**実装計画に影響する質問**

これらの質問への回答は、詳細な実装計画やスケジュールに影響します。

**主なカテゴリ**:
- チェック作業のボトルネック
- チェックリストの更新頻度
- 図面のサイズ
- 数値計算の必要性
- 複数図面比較の必要性
- 処理速度の目標
- OCR処理の速度要件
- 図面要素認識の精度要件・実装方法
- 過去データの学習の必要性
- データの形式・準備時間
- マニュアルのデジタル化状況
- 運用コストの内訳
- 投資対効果の目標
- リスク対応
- 業務フローの変更許容度
- データ保存期間
- アクセス制御要件
- 機能の優先順位（後回しにできる機能）
- MVPの最小機能・完成時期

### 低優先度 - 6問
**参考程度の質問**

これらの質問への回答は、参考情報として有用ですが、実現可能性評価には直接影響しません。

**主なカテゴリ**:
- 過去データの形式
- 学習用データの準備時間
- マニュアルの更新頻度
- メンテナンス・改善の予算
- 教育・トレーニングの時間
- 一時的な業務効率の低下の許容度

---

## 回答の進め方

### Step 1: 最優先（必須）の質問に回答
まず、最優先（必須）の8問に回答してください。これらが回答できないと、実現可能性の評価ができません。

### Step 2: 高優先度の質問に回答
次に、高優先度の23問に回答してください。これらは技術選定や開発計画に大きく影響します。

### Step 3: 中優先度の質問に回答
中優先度の24問に回答して、詳細な実装計画を立てます。

### Step 4: 低優先度の質問に回答（任意）
低優先度の6問は、時間があるときに回答してください。

## 特に重要な質問（最優先 + OCR関連の高優先度）

実現可能性評価において、特に重要な質問を以下に示します：

1. **Q2-1, Q2-2**: 図面の形式と割合 → OCRの必要性を判断
2. **Q3-1**: テキスト検索で対応できる割合 → 技術的難易度を判断
3. **Q5-1**: OCR機能の必須性 → OCRの必要性を判断
4. **Q18-1, Q18-2**: スキャン図面の割合とOCR精度 → 実現可能性を判断
5. **Q10-1, Q10-2**: 開発予算・期間 → 実現可能性を判断

これらの質問に回答できれば、大まかな実現可能性の評価が可能です。

## CSVファイルの使い方

1. **優先順位でソート**
   - スプレッドシートで「優先順位」列でソート
   - 最優先（必須）→ 高優先度 → 中優先度 → 低優先度の順に並べ替え

2. **フィルタリング**
   - 最優先（必須）のみを表示して回答
   - 次に高優先度のみを表示して回答

3. **回答欄に記入**
   - 「回答欄」列に回答を記入
   - 選択肢がある場合は該当する選択肢を記入

4. **優先順位理由を参照**
   - 「優先順位理由」列で、なぜその質問が重要かを確認
