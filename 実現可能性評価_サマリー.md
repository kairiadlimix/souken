# 図面チェックAIシステム 実現可能性評価 サマリー

## 重要なポイント

### 1. OCR部分の研究的要素について

**現状の認識**
- OCR部分は研究的要素が強く、実現可能性が不確実
- 特に日本語図面のOCRは技術的難易度が高い
- スキャン図面の品質によって精度が大きく変動

**推奨アプローチ**
1. **段階的実装**: まずテキストベース（CAD出力PDF）から開始
2. **OCR技術の選定**: 商用OCR API（Google Cloud Vision等）を優先検討
3. **代替案の準備**: OCRが困難な場合は設計事務所にCAD出力PDFの提出を依頼

### 2. 実現可能性の判断基準

#### 実現可能性: 高 ✅
- スキャン図面の割合が30%未満
- チェック項目の80%以上がテキスト検索で対応可能
- OCR不要、または商用OCR APIの利用が可能

#### 実現可能性: 中 ⚠️
- スキャン図面の割合が30-50%
- チェック項目の50-80%がテキスト検索で対応可能
- OCRが必要だが、商用OCR APIの利用が可能

#### 実現可能性: 低 ❌（研究的要素が高い）
- スキャン図面の割合が50%以上
- チェック項目の50%未満がテキスト検索で対応可能
- OCR精度が低く、自社開発が必要

### 3. 開発マイルストーン

```
Phase 1: MVP（テキストベース）     [2-3ヶ月] 実現可能性: 高
    ↓
Phase 2: OCR対応                  [2-3ヶ月] 実現可能性: 中-高
    ↓
Phase 3: 高度な機能               [6-8ヶ月] 実現可能性: 低-中
    ↓
Phase 4: 最適化・改善              [継続的]  実現可能性: 高
```

### 4. 重要な質問（優先度順）

#### 最優先質問
1. **スキャン図面の割合は？** → OCRの必要性を判断
2. **チェック項目のうち、テキスト検索で対応できるものは？** → 実現可能性を判断
3. **OCR精度の要件は？** → 技術選定を判断
4. **開発予算・期間の上限は？** → 実装計画を判断

#### 次点質問
5. 現在のチェック作業の工数・コスト
6. 図面の品質（解像度、鮮明度）
7. 過去データ・マニュアルの有無
8. システム導入による業務フロー変更の許容度

### 5. OCR技術の選択肢

| 選択肢 | 実現可能性 | コスト | 精度 | 開発期間 | リスク |
|--------|-----------|--------|------|----------|--------|
| オープンソース（Tesseract） | 中 | 無料 | 中（日本語は弱い） | 2-3ヶ月 | 中 |
| 商用OCR API | 高 | 月額数万円〜 | 高（日本語対応が良い） | 1-2ヶ月 | 低 |
| 自社開発 | 低 | 高 | 不明 | 6-12ヶ月以上 | 高 |

**推奨**: 商用OCR APIを優先検討。コストが許容できない場合はオープンソースを試し、必要に応じて商用APIに移行。

### 6. リスク管理

#### 高リスク項目
1. **OCR精度が期待通りでない**
   - 対策: 商用OCR APIの利用、手動確認の併用
   - 代替案: 設計事務所にCAD出力PDFの提出を依頼

2. **図面要素認識の実現が困難**
   - 対策: Phase 3はオプションとして位置づけ
   - 代替案: テキストベースのチェックに集中

#### 中リスク項目
1. **チェック項目の精度が低い**
   - 対策: 実際の図面でテスト、ルールの改善

2. **開発期間・予算の超過**
   - 対策: 段階的な実装、優先順位の明確化

### 7. 推奨されるアプローチ

#### アプローチ1: 段階的実装（推奨）⭐
1. **Phase 1**: テキストベースのチェックのみ（実現可能性: 高）
2. **Phase 2**: OCR機能の追加（実現可能性: 中-高）
3. **Phase 3**: 高度な機能（実現可能性: 低-中、オプション）

#### アプローチ2: 代替案の検討
- 設計事務所への依頼: CAD出力PDFの提出を必須化
- 手動チェックの併用: スキャン図面は手動チェック
- ハイブリッド方式: 自動チェック + 手動確認

## まとめ

このプロダクトの実現可能性は、**スキャン図面の割合とOCR技術の選定**に大きく依存します。

**推奨される進め方**:
1. まず質問リストに回答して現状を把握
2. Phase 1（テキストベース）から開始
3. OCRが必要な場合は商用OCR APIを検討
4. 段階的に機能を拡張

**重要な判断ポイント**:
- スキャン図面の割合が50%以上 → OCR必須、研究的要素が高い
- チェック項目の50%未満がテキスト検索で対応可能 → 実現可能性が低い
- OCR精度が低い場合の代替案を準備 → 設計事務所への依頼、手動チェックの併用
